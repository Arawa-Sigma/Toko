# Product Requirements Document (PRD)
## SembakoBerkah — Platform Toko Sembako Online

---

## 1. Ringkasan Produk

| Atribut | Detail |
|---|---|
| **Nama Produk** | SembakoBerkah |
| **Tipe** | Web App E-Commerce (Next.js 16 + Supabase) |
| **Target Pengguna** | Pelanggan, Staff Gudang, Owner Toko |
| **Platform** | Web (Mobile-first, Responsive) |
| **Tech Stack** | Next.js 16, React 19, Supabase (PostgreSQL + Auth), Zustand, Leaflet Maps |

---

## 2. Tujuan Produk

SembakoBerkah adalah platform e-commerce untuk toko sembako lokal yang memungkinkan pelanggan berbelanja secara online, memantau pesanan, dan mengajukan retur — sementara owner dapat mengelola seluruh operasional toko dari satu dashboard terpadu.

**Masalah yang Diselesaikan:**
- Pelanggan tidak perlu datang ke toko untuk beli sembako
- Owner dapat memantau stok, keuangan, dan pesanan secara real-time
- Staff gudang bisa update status pengiriman secara digital

---

## 3. Aktor / Pengguna

| Role | Deskripsi | Akses |
|---|---|---|
| **Pelanggan** | Pengguna umum yang berbelanja | Etalase, Keranjang, Checkout, Profile, Retur |
| **Staff Gudang** | Karyawan yang mengelola pengiriman | Halaman Pengiriman `/pengiriman` |
| **Owner** | Pemilik toko | Dashboard penuh: Produk, Keuangan, Pesanan, Diskon, Users, Chat, Notifikasi, Retur |
| **All-Role** | Mode khusus Owner untuk coba semua fitur | Semua akses di atas sekaligus |

## 6. Database Schema (Supabase / PostgreSQL)

```
products         → id, name, slug, category, price, stock, image_url, status,
                   discount, discount_end_date, is_best_seller, variants (JSON)

orders           → id, user_id, buyer_name, courier, total_amount, status,
                   payment_method, created_at

order_items      → id, order_id, product_id, quantity, price_at_purchase

profiles         → id (= auth.users.id), full_name, username, phone, avatar_url,
                   gender, birth_date, role

addresses        → id, user_id, name, phone, region, street, detail, label,
                   is_primary, coordinates (JSON)

vouchers         → id, code, discount_type (percent/flat), discount_amount,
                   min_order, max_discount, is_active

notifications    → id, title, message, type, unread, created_at

returns          → id, order_id, user_id, reason, photo_url, status, created_at

expenses         → id, description, amount, created_at
```

## 7. Arsitektur Teknis

```
Frontend (Next.js 16 App Router)
├── Layout Global  → Navbar, Footer, Toast, LayoutWrapper
├── State Management → Zustand (cart, session, wishlist, voucher)
├── Auth → Supabase Auth (Email + Google OAuth)
├── Database → Supabase PostgreSQL (via @supabase/ssr)
└── Storage → Supabase Storage (gambar produk & avatar)
```
